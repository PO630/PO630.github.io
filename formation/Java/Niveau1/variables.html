Déclarez des Variables

Les variables permettent de stocker des données.

    Une variable est définie par :
  • Un type : Le type de données qu'elle peut contenir (ex. int, String).
  • Un nom : Un identifiant unique.
  • Une valeur : Optionnel, mais elle peut être initialisée au moment de la déclaration.

/** Syntaxe générale : */
type nomVariable = valeurInitiale;

/** Exemple : */
int age = 25;  // Variable de type entier initialisée à 25
String nom = "Alice";  // Variable de type chaîne de caractères

Types primitifs

Les types primitifs sont les blocs de base de données en Java. Ce sont des types simples intégrés au langage, utilisés pour stocker des valeurs directes (comme des nombres ou des caractères).

  • Ils sont définis directement dans le langage.
  • Ils ne peuvent pas être null (sauf s'ils sont utilisés comme objets dans un type enveloppe, comme Integer pour int).
  • Ils ont une taille et une structure fixes.
  • Ils ne possèdent pas de méthodes associées.
TypeTailleDescriptionExemple
byte1 octet Entier de petite taille (-128 à 127) byte b = 100;
short2 octets Entier (-32,768 à 32,767) short s = 20000;
int4 octets Entier (-2^31 à 2^31-1) int i = 123456;
long8 octets Entier de grande taille (-2^63 à 2^63-1) long l = 123456789L;
float4 octets Nombre décimal simple précision float f = 3.14f;
double8 octets Nombre décimal double précision double d = 3.141592;
char2 octets Caractère Unicode char c = 'A';
boolean1 bit Valeur booléenne (true ou false) boolean b = true;

Types Non Primitifs

Les types non primitifs sont des types complexes, également appelés référencés, car ils stockent une référence à l'objet dans la mémoire, et non une valeur directe.

  • Ils peuvent être créés par le programmeur (exemple : classes, tableaux).
  • Ils peuvent être null.
  • Ils possèdent des méthodes associées.
  • Ils sont souvent plus lourds que les types primitifs.
TypeExemple
String (Chaîne de caractères) String nom = "Java";
Tableaux int[] nombres = {1, 2, 3, 4};
