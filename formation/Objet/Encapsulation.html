L'Encapsulation

L'encapsulation est un concept fondamental de la programmation orientée objet qui permet de restreindre l'accès direct à certains composants d'un objet. Ce principe vise à protéger les données internes d'une classe contre des modifications non autorisées, tout en fournissant une interface claire et contrôlée pour interagir avec ces données.

Interface : La boite noir

L'idée principale de l'encapsulation est de cacher l'état interne d'un objet et de ne permettre son accès qu'à travers des méthodes spécifiques, souvent appelées méthodes d'accès ou getters et setters. Ces méthodes permettent de contrôler comment les attributs d'un objet sont modifiés ou récupérés. Par exemple, un setter peut inclure des vérifications pour s'assurer que les valeurs assignées à un attribut sont valides.

Une interface d’objet agit comme un ensemble de commandes qu’on peut utiliser pour interagir avec l’objet, un peu comme les boutons d’une boîte noire. L’utilisateur sait quelles commandes sont disponibles, mais pas comment elles fonctionnent en interne. Cela permet de se concentrer sur l’utilisation de l’objet sans avoir besoin de comprendre son fonctionnement interne, ce qui simplifie l’interaction et protège les données internes.

L'interface d'un objet ne vise pas seulement à restreindre l'accès aux données, mais surtout à assurer la stabilité et l'intégrité du code. Cela permet que même si la logique interne de l’objet change (comme une amélioration de calcul ou un changement dans la manière dont les données sont stockées), l'interface elle-même reste inchangée.

Ainsi, les autres parties du programme qui utilisent cet objet peuvent continuer à fonctionner sans être impactées par les modifications internes.

Mise en place

Dans la programmation orientée objet, les mots-clés public et privé (private) définissent la visibilité ou le niveau d'accès des attributs et des méthodes d'une classe.

  • Public : Un attribut ou une méthode déclarée comme public est accessible depuis n'importe quelle autre classe. Cela signifie que toute partie du programme peut interagir directement avec cet attribut ou cette méthode.
  • Privé : Un attribut ou une méthode déclarée comme private est uniquement accessible depuis la classe elle-même. Cela permet de contrôler l'accès aux données sensibles ou de les protéger contre des modifications non souhaitées. Avec private, nous pouvons garantir que certaines valeurs ne changent que par des moyens sûrs et contrôlés, comme par un setter.
  • Protected : permet à une variable ou une méthode d’être accessible dans sa propre classe, ses sous-classes (dans le même package ou un autre) et les autres classes du même package. Cela offre un niveau d'accès entre private et public, utile pour créer des objets qui exposent certains aspects à leurs sous-classes sans tout rendre accessible publiquement. 


/** Exemple JAVA */
public class Horloge 
{
	public String nom; // Attribut public, accessible directement
    private int heure; // Attribut privé

	// Getters et setters
    public int getHeure() {
        return heure;
    }

	public void setHeure(int heure) {
        this.heure = heure % 24; // Limite l'heure à une valeur valide (0-23)
    }
}


# Exemple PYTHON
class Horloge:
	def __init__(self, nom, heure=0, minute=0, seconde=0):
			self.nom = nom  # Attribut public
			self._heure = heure % 24  # Attribut privé
			self._minute = minute % 60  # Attribut privé
			self._seconde = seconde % 60  # Attribut privé

		# Getters et setters
		def get_heure(self):
			return self._heure
		
		def set_heure(self, heure):
			self._heure = heure % 24

En Python, il n’y a pas de mot-clé strictement équivalent pour les niveaux de visibilité comme en Java, mais par convention, les attributs privés sont préfixés d’un underscore _.

Exemple

Maintenant, nous allons changer le fonctionnement interne de notre objet Horloge sans en modifier l'interface. Pour optimiser notre code, nous n'utiliserons plus qu'un seul attribut, secondes, au lieu des trois attributs heure, minute et seconde.

Cette approche simplifie la gestion du temps tout en conservant les mêmes méthodes setHeure, setMinute, et setSeconde. L’utilisateur de l’objet pourra continuer à utiliser l’horloge de la même manière, sans se soucier des changements internes, qui demeurent invisibles. 


/** Exemple JAVA */
public class Horloge {
    private int secondes; // On stocke tout en secondes maintenant
    public String nom;

    public Horloge(int heure, int minute, int seconde, String nom) {
        this.nom = nom;
        setHeure(heure);
        setMinute(minute);
        setSeconde(seconde);
    }

    // Ajuste les heures en secondes
    public void setHeure(int heure) {
        this.secondes += heure * 3600;
    }

    // Ajuste les minutes en secondes
    public void setMinute(int minute) {
        this.secondes += minute * 60;
    }

    // Ajuste les secondes directement
    public void setSeconde(int seconde) {
        this.secondes += seconde;
    }

    // Récupère l'heure au format HH:MM:SS
    public String getFormattedTime() {
        int heures = secondes / 3600;
        int minutes = (secondes % 3600) / 60;
        int sec = secondes % 60;
        return String.format("%02d:%02d:%02d", heures, minutes, sec);
    }
}
